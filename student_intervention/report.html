Preprocess Feature Columns¶

As you can see, there are several non-numeric columns that need to be converted! Many of them are simply yes/no, e.g. internet. These can be reasonably converted into 1/0 (binary) values.

Other columns, like Mjob and Fjob, have more than two values, and are known as categorical variables. The recommended way to handle such a column is to create as many columns as possible values (e.g. Fjob_teacher, Fjob_other, Fjob_services, etc.), and assign a 1 to one of them and 0 to all others.

These generated columns are sometimes called dummy variables, and we will use the pandas.get_dummies() function to perform this transformation. Run the code cell below to perform the preprocessing routine discussed in this section.

In [6]:
def preprocess_features(X):
    ''' Preprocesses the student data and converts non-numeric binary variables into
        binary (0/1) variables. Converts categorical variables into dummy variables. '''
    
    # Initialize new output DataFrame
    output = pd.DataFrame(index = X.index)

    # Investigate each feature column for the data
    for col, col_data in X.iteritems():   
        # If data type is non-numeric, replace all yes/no values with 1/0
        if col_data.dtype == object:
            col_data = col_data.replace(['yes', 'no'], [1, 0])

        # If data type is categorical, convert to dummy variables
        if col_data.dtype == object:
            # Example: 'school' => 'school_GP' and 'school_MS'
            col_data = pd.get_dummies(col_data, prefix = col)  
        
        # Collect the revised columns
        output = output.join(col_data)
    
    return output

Answer:

  1. Logistic Regression

    • Applications:
      • Logistic regression was used to predict the outcome of whether or not an accepted student will enroll at a school (http://www.math.montana.edu/grad/writing-projects/09sulock.pdf)
      • The intensity of ischemic heart disease risk factors in rural areas of the Western Cape, South Africa was studied using a logistic-regression model (Hastie, T. and Tibshirani, R. (1987). Nonparametric logistic and proportional odds regression, Applied Statistics 36: 260276)
    • Strengths
      • Simple to interpret the model
      • Not susceptible to high variance (esp. if used with regularization)
      • Vanilla model does not have any hyperparameters for the user to provide.
      • Logistic regression is faster when the dimensionality is large
    • Weakness:
      • Logistic regression works better if there's a single linear decision boundary. For example, it would not do well if the decision surface is parallel to two axes in a 2D classification case (a right-angle decision boundary).
    • Why use this: Logistic regression is a pretty well-behaved classification algorithm that can be trained as long as features are roughly linear and the problem to be linearly separable. We expect this to be the case with the given data.

  2. SVM Classifier

    • Applications: SVM has been used to classify persons with and without common diseases, specifically to detect persons with diabetes and pre-diabetes in a cross-sectional representative sample of the U.S. population. Reference: Wei Yu, et. al., BMC Medical Informatics and Decision Making 2010
    • Strengths
      • Training is relatively easy
      • Does not get stuck in local optimal
      • It scales relatively well to high dimensional data
      • Tradeoff between classifier complexity and error can be controlled explicitly
      • Non-traditional data like strings and trees can be used as input to SVM, instead of feature vectors
    • Weaknesses
      • Need a good kernel function (requires domain knowledge)
      • Need to provide values for hyperparameters (C, kernel function)
      • They are inefficient to use for big datasets
      • Tricky to extend this to multi-class classification
    • Why use this: SVM classifiers are mostly quite accurate and allow easy addition of kernels to tranform the feature to a non-linear space. Our current problem is relatively small and would not take too long with SVM.

  3. Random Forest Classifier (RF)

    • Applications: A big data Airline dataset was used with Random Forests to predict if a flight was likely to arrive on time or with a delay larger than 15 minutes. Reference: Random Forests for Big Data Robin Genuer et. al. (https://arxiv.org/pdf/1511.08327.pdf)
    • Strengths:
      • Typically perform well esp. if enough number of ensembles are used.
      • They do not expect linear features or even features that interact linearly.
      • Categorical variables, theoretically, don't have to be dummy encoded (depends on the implementation).
    • Weaknesses:
      • Decision trees (the weak learners used in RF) treat each feature independently and divide the feature space parallel to the axes. This leads to poor results if the data points show a correlation between features or if they're non-linearly distributed.
      • Decision trees themselves are prone to overfitting. Random forests reduce that by averaging over multiple trees, but can overfit if we don't have enough ensembles.
      • Need to provide value for a hyperparameter (number of trees).
      • Difficult to interpret the model since there are multiple trees
    • Why use this: RFs can almost work "out of the box", hence are very popular. They are easy to use and could work well with our mostly categorical data.

def train_classifier(clf, X_train, y_train):
    ''' Fits a classifier to the training data. '''
    
    # Start the clock, train the classifier, then stop the clock
    start = time()
    clf.fit(X_train, y_train)
    end = time()
    
    # Print the results
    print "Trained model in {:.4f} seconds".format(end - start)

    
def predict_labels(clf, features, target):
    ''' Makes predictions using a fit classifier based on F1 score. '''
    
    # Start the clock, make predictions, then stop the clock
    start = time()
    y_pred = clf.predict(features)
    end = time()
    
    # Print and return results
    print "Made predictions in {:.4f} seconds.".format(end - start)
    return f1_score(target.values, y_pred, pos_label='yes')


def train_predict(clf, X_train, y_train, X_test, y_test):
    ''' Train and predict using a classifer based on F1 score. '''
    
    # Indicate the classifier and the training set size
    print "Training a {} using a training set size of {}. . .".format(clf.__class__.__name__, len(X_train))
    
    # Train the classifier
    train_classifier(clf, X_train, y_train)

    print "F1 score for training set: {:.4f}.".format(predict_labels(clf, X_train, y_train))
    print "F1 score for test set: {:.4f}.".format(predict_labels(clf, X_test, y_test))

In one to two paragraphs, explain to the board of directors in layman's terms how the final model chosen is supposed to work. Be sure that you are describing the major qualities of the model, such as how the model is trained and how the model makes a prediction. Avoid using advanced mathematical or technical jargon, such as describing equations or discussing the algorithm implementation.

Answer:

SVM classifier separates data into two classes by computing a hyper-plane between the classes. This hyper plane is computed such that the distance (called margin) between plane and the closest points (called support vectors) on both sides is maximized. These support vectors have direct bearing on the optimum location of the decision surface and depending on the parameters of the classifier, we can also have some support vectors on the wrong side of the classifying plane.

SVM uses a tuning parameter (C), which weights training classification errors and thus controls the generalization ability. The higher the value of C, the higher is the penalty to missclassified point thus producing a lower generalization. Poor generalization implies the training score would be good but the test score will be bad. By choosing a low C, we reduce the overfitting but could also result in lower quality model.

The model make a prediction by computing on which side of the classifying plane a test point lies.
